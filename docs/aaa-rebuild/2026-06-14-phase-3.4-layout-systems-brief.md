## 1. The three sub-items (and recommended order)

> 3.4 = **constraints · component variants · non-lossy instance overrides.** Recommended order is **constraints → overrides → variants** (ascending risk: constraints is isolated; overrides fixes real data loss and is the substrate variants needs; variants is the biggest and builds on overrides).

### Relevant code map (read these before touching anything)

- **`src/apps/xdesign/store.ts`** — the model.
  - `ShapeBase` already has `parentId`, `isMain?`, `linkedMainId?`, and auto-layout fields `layoutSizingH/V` (`fixed|hug|fill`), `layoutPositioning` (`auto|absolute`). `FrameShape` has `layoutMode` (`none|horizontal|vertical`), padding, `primaryAxisAlign`/`counterAxisAlign`.
  - Component actions: `createInstance(mainId, at)` (deep-clones main's subtree, stamps `linkedMainId` on every node, `isMain:false`), `syncFromMain(instanceId)` (**re-clones from main — DROPS all instance overrides**, the lossy behavior to fix), `detachInstance` (clears `linkedMainId` on the subtree), `toggleMainComponent`, `reparent` (cycle-safe).
  - History: `pushHistory()` once before a mutation = one undo step. `patchMany(ids, fn)` / `updateShape(id, patch)` (note: `updateShape` with `x`/`y` cascades the delta to descendants — frames carry children).
- **`src/apps/xdesign/Canvas.tsx`** — interaction.
  - `onHandleMouseDown` builds the resize `startShapes` map **from `selection` only** → resizing a frame currently resizes just the frame box; **freeform children do NOT reflow** (greenfield for constraints). The resize math is in the move-effect `drag?.kind === "resize"` branch (proportional `relX/relY/relW/relH` replay against the live bbox).
  - `computeAutoLayout(shapes)` (`autoLayout.ts`) → `LayoutOverrides` Map applied at render in `displayShapes`. Auto-layout frames position flowed children here; **constraints are the complement for `layoutMode: none/undefined` frames** — don't run both on the same frame.
- **`src/apps/xdesign/Inspector.tsx`** — the single-select property panel + the existing component section (toggle main / create instance / sync / detach). Constraint controls + variant-property controls land here.

### Sub-item A — Constraints (pin / center / scale)

- **Model (additive, optional):** add to `ShapeBase`
  `constraintH?: "left" | "right" | "left-right" | "center" | "scale"` and
  `constraintV?: "top" | "bottom" | "top-bottom" | "center" | "scale"`.
  Default (unset) = `left`/`top` (Figma default).
- **Pure core (testable):** `applyConstraints(child, oldFrame, newFrame) → { x, y, w, h }` in a new `constraints.ts`. Semantics per axis (H shown; V symmetric):
  - `left`: keep left offset → `x` unchanged.
  - `right`: keep right offset → `x = newFrame.right - (oldFrame.right - child.x)`… i.e. pin the gap on the right; width unchanged.
  - `left-right`: pin both gaps → width stretches: `w = newFrame.w - leftGap - rightGap`.
  - `center`: keep center ratio → recenter against new bounds; width unchanged.
  - `scale`: scale offset *and* size by `newFrame.w / oldFrame.w`.
- **Wire into resize:** when the resized shape is a frame with `layoutMode` none/undefined, after each move compute every **direct child** via `applyConstraints` against the frame's old→new bounds and `patchMany` them. Must recurse (a child frame resizes → its children reflow). Auto-layout frames skip this (they use `computeAutoLayout`).
- **UI:** the Figma "pin box" widget in the Inspector single-select panel (only meaningful when the shape has a parent frame).
- **Tests:** `applyConstraints` for every H×V combo under width-only, height-only, and both-axis changes; verify `center` and `scale` arithmetic exactly.

### Sub-item B — Non-lossy instance overrides

- **Problem:** `syncFromMain` re-clones from main and **wipes local edits** (documented in `store.ts` + CLAUDE.md "simple sync model — no per-property override tracking yet").
- **Approach:** track overrides on the **instance root**, keyed by the **corresponding main-descendant id** (stable: both `createInstance` and `syncFromMain` build an `idMap` from main ids, so the main id is the durable identity across re-clones).
  - Add `overrides?: Record<string /*mainDescendantId*/, Partial<Shape>>` to the instance root (or a sibling store map keyed by instanceRootId).
  - **Capture:** when `updateShape` (or inspector edit) targets a shape with `linkedMainId`, diff the patched props vs the corresponding main descendant and record the patch under that main id. Decide the overridable prop set (fill/text/position-within-frame/opacity/etc.); never override structural fields (`parentId`, `linkedMainId`, `id`).
  - **Re-apply:** in `syncFromMain`, after building `fresh[]`, walk `idMap` (mainId→newId) and merge any recorded override patch onto the matching fresh node. Main changes win for non-overridden props; the override wins for the props the user touched.
- **UX:** a per-instance **"Reset overrides"** action (and per-prop reset is a nice-to-have). `detachInstance` still clears everything.
- **Tests (the heart):** edit an instance child → `syncFromMain` with main unchanged → override survives. Then change main → main's change applies to non-overridden props while the override still wins on its prop. Round-trip the capture→re-apply.
- **Recommend a short design pass first** (override key strategy + overridable-prop allow-list) before coding — this is the trickiest of the three.

### Sub-item C — Component variants (sets + properties)

- **Concept:** a **variant set** groups several main components, each tagged with property values; an instance picks a combination and mirrors the matching main.
- **Minimal model:**
  - A set = a container (frame) flagged `isVariantSet?: true`; its child mains each carry `variantProps?: Record<string,string>` (e.g. `{ State: "hover", Size: "lg" }`). Property *names* + allowed *values* are derived from the children.
  - Instance gains `variantSelection?: Record<string,string>`. Resolving = find the set member whose `variantProps` matches the selection; swap the instance to mirror that main (reuse the `syncFromMain` clone path) **while preserving position + overrides** (depends on Sub-item B).
- **UI:** when a selected instance belongs to a set, the Inspector shows a dropdown per property; changing one re-resolves + swaps the variant.
- **Tests:** selection→matching-main resolution; missing-combination fallback (nearest / first).
- **Biggest piece — its own design pass.** Build it last so it can lean on B's override machinery.

## 4. Gotchas

- Don't run constraints **and** auto-layout on the same frame — `layoutMode` none/undefined → constraints; horizontal/vertical → `computeAutoLayout`.
- Resize `startShapes` is selection-only today; constraints must reach **unselected children** of the resized frame (and recurse).
- Override identity must survive a full re-clone → key by **main-descendant id**, not the instance's own (regenerated) ids.
- `updateShape` cascades x/y to descendants — keep that in mind when capturing position overrides.
- Keep every new `Shape` field **optional** so existing `xdesign.doc` blobs hydrate unchanged (back-compat = the migration-free win).
